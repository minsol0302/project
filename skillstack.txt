[STEP 2] 백엔드 Python 환경 세팅
-------------------------------------------------------
(2-1) conda 가상환경 생성 및 활성화

(2-2) 패키지 설치
  cd C:\Users\hi\Documents\www.wouldyouin.com\api.wouldyouin.com
  pip install -r requirements.txt

(2-3) .env 파일 생성 (절대 git에 올리면 안됨)


  .env 파일 내용 (아래 그대로 복사 후 실제 값 입력):
  ----------------------------------------------------------
  NEON_DB_URL=postgresql://유저명:비밀번호@호스트/DB명?sslmode=require
  NEON_DB_USERNAME=유저명
  NEON_DB_PASSWORD=비밀번호
  JWT_SECRET=랜덤문자열(예:wouldyouin-secret-2026)
  JWT_ALGORITHM=HS256
  JWT_EXPIRE_DAYS=7
  GEMINI_API_KEY=Gemini_API_KEY값
  CORS_ORIGINS=["http://localhost:3000","https://www.wouldyouin.com","https://wouldyouin.com"]
  ----------------------------------------------------------

  ※ NEON_DB_URL 확인 위치: https://console.neon.tech
      -> 프로젝트 선택 -> Connection string 복사
  ※ GEMINI_API_KEY 발급: https://aistudio.google.com/app/apikey

(2-4) 로컬 실행 확인
  uvicorn main:app --reload --port 8080

  브라우저에서 http://localhost:8080/health 접속
  -> {"status":"ok"} 나오면 정상

-------------------------------------------------------
[STEP 4] GitHub Actions CI/CD 시크릿 등록
         (백엔드 자동 배포 - git push 하면 EC2에 Docker로 배포됨)
-------------------------------------------------------
위치: https://github.com/000000000000000000000
     -> Settings -> Secrets and variables -> Actions -> New repository secret




등록해야 할 시크릿 목록 (이름 한 글자도 틀리면 안됨):

  시크릿 이름          | 입력할 값
  ---------------------------------------------------------------------------
  DOCKERHUB_USERNAME  | Docker Hub 아이디 (예: jiminsong02)
  DOCKERHUB_TOKEN     | Docker Hub Access Token
                      | 발급: hub.docker.com -> Account Settings
                      |       -> Security -> New Access Token
  EC2_HOST            | EC2 퍼블릭 IPv4 주소 (예: 13.125.xxx.xxx)
                      | 확인: AWS 콘솔 -> EC2 -> 인스턴스 -> 퍼블릭 IPv4 주소
  EC2_USERNAME        | ubuntu  <- 이 값 그대로 입력
  EC2_SSH_KEY         | .pem 파일 전체 내용
                      | (-----BEGIN RSA PRIVATE KEY----- 첫 줄부터 마지막 줄까지 전부)
  NEON_DB_UR          | Neon DB 연결 문자열 전체
                      | ※ 주의: 이름이 NEON_DB_UR 임 (URL 아님, L 없음!)
                      |   deploy.yml 60번째 줄에서 이 이름 그대로 사용 중
  NEON_DB_USERNAME    | Neon DB 유저명
  NEON_DB_PASSWORD    | Neon DB 비밀번호
  JWT_SECRET          | JWT 시크릿 값 (백엔드 .env의 JWT_SECRET과 동일값)
  GEMINI_API_KEY      | Gemini API Key
  ---------------------------------------------------------------------------

  ※ 시크릿 등록 완료 후, git push 하면 자동으로 EC2에 배포됨
  ※ 수동 실행: GitHub -> Actions 탭 -> 워크플로우 선택 -> Run workflow 버튼

-------------------------------------------------------
[STEP 6] DB 스키마 초기화 (테이블이 없을 때만 실행)
-------------------------------------------------------
새 Neon DB 사용 시 또는 테이블이 없을 때만 실행.
반드시 EC2 Docker 컨테이너 안에서 실행해야 함 (로컬 아님).

  (6-1) EC2 SSH 접속 (로컬 PowerShell에서 실행)
    ssh -i "키파일이름.pem" ubuntu@EC2_퍼블릭IP

  (6-2) 실행 중인 컨테이너 이름 확인
    docker ps

  (6-3) 컨테이너 내부에서 init_db.py 실행
    docker exec -it vding-api python3 scripts/init_db.py

  ※ "Table created" 또는 "already exists" 메시지 나오면 정상
  ※ Neon DB 콘솔(https://console.neon.tech)에서 테이블 생성 확인 가능
     -> Tables 탭 -> vding_users, vding_projects 테이블 보이면 OK



-------------------------------------------------------
[STEP 7] 가비아 DNS 설정 (도메인 연결)
-------------------------------------------------------
위치: https://www.gabia.com -> 로그인 -> My가비아 -> DNS 설정 -> wouldyouin.com

  호스트명   | 타입  | 값
  ----------------------------------------------------------
  www        | CNAME | cname.vercel-dns.com.
  api        | A     | EC2 퍼블릭 IPv4 주소 (숫자 IP)
  ----------------------------------------------------------

  ※ EC2 재시작 시 퍼블릭 IP가 변경될 수 있음
    바뀌면 가비아 A레코드 IP 즉시 업데이트해야 함
  ※ TTL은 기본값(600) 유지
  ※ api.wouldyouin.com 은 Vercel에 추가하면 안됨 (EC2 전용)
